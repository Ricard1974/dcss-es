====================================================================

Este documento explica todas las opciones en la última versión de Dungeon
Crawl Stone Soup. Si te atascas o algunas cosas no parecen funcionar correctamente,
pedir ayuda en cualquiera de los recursos de discusión comunitaria descritos
http://crawl.develz.org/.

------

0- Generalidades en opciones.
================================================================================================================================================================================================================================================================

La forma estándar de ajustar una opción desde el default es manualmente
establecerlo en su archivo de opciones; ver (0-a) y (0-c) a continuación para detalles. Un pequeño
el número de opciones se guardan automáticamente de sus valores actuales en el
interfaz de usuario relevante en el juego; estos están marcados con "(persistente)" en
su documentación y puede ser anulado por un ajuste manual en el
archivo de opciones.

0-a Archivos de opción estándar y adicionales.
-------

En el juego en línea, elija "edit rc" de la pantalla de selección de juegos webtiles,
o introduzca el editor de archivos rc a través de ssh.

En el juego offline, Crawl utiliza el primer archivo de la siguiente lista como su opción
archivo:

* Ajustes/init.txt
* init.txt (en el directorio Crawl)
* .crawlrc (en el directorio de Unix)

Alternativamente, puede utilizar la opción de línea de comandos
-rc
que le dice a Crawl que use precisamente ese archivo.

En la pantalla inicial, Crawl te dice qué archivo de entrada utiliza, si lo hay. El
La ubicación del directorio Crawl es específica para el sistema.

Es posible incluir archivos adicionales dentro del archivo de opciones a través de
incluir =
Tenga en cuenta que estos se tratan como si su contenido fuera introducido verbatim
en el archivo de opciones. Por defecto, Crawl busca todos estos incluidos
archivos en la configuración / directorio pero puede especificar una carpeta especial
usando la opción de línea de comandos
-rcdir

Los archivos de opción externa tienen varios usos: Permiten una rápida personalización
de la apariencia general de Crawl (coloura / supresión de mensajes, y
el estilo de inventario). Más elaborada opción magia (generalmente conteniendo
lua) también se puede ocultar convenientemente de esta manera. Por último, hay algunos
archivos de opción adicionales en la configuración/ directorio; estos permiten establecer
algunas opciones y glifos monstruo a sus defectos de versiones anteriores.
Vea el encabezado del init.txt predeterminado para más detalles.

Los siguientes archivos se incluyen automáticamente con Crawl para uso fácil, y
se puede incluir sin ninguna otra configuración:

* advanced optioneering.txt
* dvorak command keys.txt
* colemak command keys.txt
* neo command keys.txt
* no vi command keys.txt
* safe move shift.txt

0-b Opciones en la línea de comandos.
--------------------

Una manera rápida de hacer pequeños cambios en las opciones, sin tener que
cambiar a un archivo de opción diferente, es utilizar las opciones de línea de comando
-extra-opt-first or -extra-opt-last, which make it as if the given
La opción era (respectivamente) en la parte superior o inferior del archivo de opción. Para
ejemplo,

-extra-opt-last wiz mode=yes

hará que cualquier nuevo juego comience en modo mago. -extra-opt-first and
-extra-opt-last se puede utilizar varias veces en la misma línea de comandos.

0-c Opciones y cómo configurarlas.
--------------------

Hay tres tipos amplios de opciones de Crawl: valores verdaderos/falsos
(booleans), valores arbitrarios y listas de valores. En este documento,
las opciones se describen generalmente con sus valores predeterminados (si hay un
por defecto); esto también debe explicar cuál de los tipos mencionados
Lo es. Cada opción debe tener algunos comentarios sobre cómo se utiliza normalmente
- pero tenga en cuenta que las opciones que desea utilizar dependen de su
jugando estilo y a veces también en su sistema operativo.

Hay dos estilos que puedes utilizar para configurar opciones. El clásico
nombre=valor sintaxis, una opción por línea:
recordar nombre = verdadero
explore greedy = false

Y la línea de opción combinada de estilo NetHack:
OPTION = remember name, !explore greedy

El segundo estilo es útil para especificar opciones simples en algunas líneas,
pero no se puede utilizar para opciones que toman listas complejas de valores
(como la opción autopickup exceptions).

Algunas opciones necesitan un camino como argumento; aquí tienes que usar un
ruta del sistema de archivos adecuado para su sistema. Otras opciones aceptan regular
expresiones (regexes): aquí simplemente puede utilizar cadenas ordinarias, adaptar
el regexes sugerido a sus necesidades o buscar en Internet para regex
sintaxis.

Tenga en cuenta que en los modos tutoriales y consejos, algunos de los ajustes de sus opciones pueden
ser sobrescrito para facilitar las explicaciones. This concerns the following
opciones, y su configuración forzada para juegos tutoriales son los siguientes:
clear messages = true
arma = hacha de mano, para Berserkers en modo de pistas

0-d Opciones de lista.
---

Varias opciones pueden tener una lista de valores. La mayoría pero no todos estos
opciones permiten establecer múltiples valores en una sola línea, separadas por comas.
Es posible restablecer, añadir artículos a, o eliminar artículos de tal opción.
Agregar valores al final de la lista (apéndice):
drop filter += inútil, encantador
Agregar valores al comienzo de la lista (prependiendo):
drop filter ^= prohibido
Eliminar los valores de la lista (exactar sólo el partido):
drop filter -= enchant
Restablecer la lista:
drop filter = bad item, dangerous item
Vacía la lista:
drop filter =

La sintaxis "listopt = valor" primero aclara la lista, luego añade valores. In
versiones anteriores de Crawl era un sinónimo para +=, y sólo desnudo
"listopt =" limpió la lista.

Para muchas opciones, como drop filter, no hay ninguna diferencia
entre el gasto y el prependimiento. Sin embargo, otras opciones, como
message colour, preocúpate del orden de los artículos; estos serán notados como
"Ordenadas opciones de lista" en sus descripciones. En la mayoría de los casos, artículos anteriores
tomar precedencia sobre los posteriores, por lo que += diferir a los partidos existentes (incluyendo
predeterminados) mientras ^= los anula. Así, por ejemplo, con la secuencia de
opciones:
message colour = cyan:hits
message colour ^= red:crimson
message colour += amarillo:killer
el mensaje "El carmesí imp golpea!" se mostrará en rojo, mientras que "El
la abeja asesina te golpea!" se mostrará en cyan.

0-e Aliases, variables, y otras directivas


Para nombres de opciones largas, puede definir alias de opción haciendo:
alias := long option name
Por ejemplo, puede usar:
ae := autopickup exceptions
y luego utilizar "ae" en lugar de "autopickup excepciones":
ae += < > >

Puede definir atajos para valores de opción (variables). Por ejemplo,
$useless := darkgrey
podría utilizarse conjuntamente con
menu colour += $useless:Aleatorio sin uso

Para evitar que una variable se cambie puede hacer que sea una constante:
constante = inútil

Esto es útil si desea evitar que un archivo incluido altere
una variable.

Véase dat/defaults/standard colours.txt para el uso de alias y variables. Nota
que cambiar las variables definidas en estos archivos sólo afectará a su propio
archivo de configuración, ya que las opciones predeterminadas ya se habrán cargado. Si
quieres cambiarlos, tendrás que anular las opciones.

Para restablecer un valor a sus defectos, puede utilizar:
default = option name

Esto no funcionará para ciertas opciones que tienen perspicacia personalizada.

1- Pantalla de inicio.
===============================================================

Las siguientes opciones son una comodidad para ayudarle a iniciar rápidamente su
juego de Crawl, y/o configurar el inicio del juego desde un archivo o el
línea de mando.

nombre = Delilah
Si está listo, ese es el nombre que todos tus personajes de Crawl recibirán. Ajuste
esto evitará el menú principal por defecto; véase `name bypasses menu`.

tipo = normal
Elige un tipo de juego. Los valores válidos son "normales", "seeded", "tutorial",
"arena", "sprint", "descent", y "hintas". Si se establece explícitamente (no sólo
izquierda al predeterminado), esto establece el tipo de juego predeterminado en el menú principal.
Si el menú principal se pasa por alto (por ejemplo, estableciendo `nombre'), esta opción puede
ser utilizado para determinar el tipo de juego. "descenso" es sólo válido en alfa
(desarrollo) construye.

recordar nombre = verdadero
Crawl recuerda las opciones (especie, fondo, etc.) que usaste
para crear tu último personaje. Tú puede reciclarlos en el
la pantalla inicial para un nuevo personaje. Si esta opción se establece
Cierto, Crawl también recordará el apellido que usaste.

Si utiliza esta opción y desea introducir un nombre  after 
elegir su especie y fondo, debe entrar . en
aviso inicial - golpear Entra en el aviso de nombre
simplemente reutiliza tu antiguo nombre si se establece el nombre.

arma += (short sword ←...
(opción de registro)
Especificar la opción de arma permite evitar el arma
Pantalla de selección. Tridents, flails, rapiers, espadas largas y guerra
los ejes están restringidos a combatientes y gladiadores, y los cuartos
sólo están restringidos a los gladiadores. El aviso de arma estándar será
muestra si se especifica una opción ilegal para el fondo seleccionado.
La opción "viable" hace una elección aleatoria entre las armas "buenas"
para el personaje elegido. Especificar más de una opción provoca la
juego para seleccionar al azar un arma de la lista dada. La opción combo
anula (y está anulado por) esta opción.

especie += (Human confidencialidad...
(opción de registro)
Las abreviaturas habituales (Hu, MD, etc.) funcionan. "viable" elegirá un
especies viables para un fondo dado si el fondo es elegido
primero. Especificar múltiples especies hace que se seleccione una en
al azar de la especie dada. La opción combo anula (y es
overridden by) esta opción.

fondo += (Fighter tención...
(opción de registro)
Aquí se pueden utilizar las abreviaturas (Fi, HW, CA, etc.). Lo mismo
caveat se aplica a "viable": sólo es realmente significativo si el
también se establece la opción de especies. Especificación de múltiples antecedentes causas
uno para ser seleccionado al azar de la especie dada. La opción combo
anula (y está anulado por) esta opción.

combo += (HuFi . corta espada ← Human Monk ← ...), ...
(opción de registro)
Especifica un conjunto completo de especies, antecedentes y (donde
aplicable) arma. Si un combo requiere una elección de arma y uno no
especificado, se solicita. Los combos pueden ser abreviados o especificados
en su totalidad. Si se especifican varios combos, se selecciona aleatoriamente
de los combos especificados. El arma, la especie y el fondo
las opciones están anuladas por (y anular) esta opción.

restart after game = tal vez/true
Cuando se establece a la verdad, en el extremo del juego, arrastrar volverá al menú principal.
Si se establece a tal vez, arrastrar volverá al menú principal sólo si la startup
las opciones no pasan el menú principal, de lo contrario saldrá. Si se establece
falso, saldrá incondicionalmente. Esta opción se establece para la verdad
azulejos locales construye por defecto, y tal vez por defecto para otras construcciones.
Esta opción es ignorada para juegos en línea.

restart after save = true
Cuando se guarda el juego, vuelva al menú principal. Esta opción
sólo tiene un efecto si reinart after game se establece a tal vez o verdad.
Esta opción es ignorada para juegos en línea.

newgame after quit = false
Cuando sea cierto, si se establece un nombre/combo, renuncia hará un reinicio rápido. En esto
configuración, con restart after game como no falso, puede ser posible
salida gate sólo a través del ahorro.

name bypasses menu = true
Cuando se establece a la verdad, tener un nombre válido establecido a través de opciones saltará el principal
menú y saltar derecho a un juego o selección de caracteres (si no hay
salvo).

default manual training = false (persistent)
Cuando se establece en la verdad, nuevos personajes comenzarán con entrenamiento de habilidad
se establece en modo manual en lugar de modo automático. Esto también establecerá el
[!] selección en el menú de habilidad (m) a "costo" por defecto, en lugar de
"entrenamiento".

autopickup starting ammo = true
Cuando se establece, al inicio del juego, el autopickup se ve forzado (como por el menú \)
para el tipo de munición inicial del jugador. Incluso cuando esta opción es
set, autopickup de esos elementos se puede deshabilitar desde el menú \ una vez
el juego ha comenzado.

game seed = ninguno
Un número que indica la semilla para inicializar el generador de números aleatorios
de. Dentro de ciertos límites, la misma semilla conducirá deterministamente
a la misma mazmorra. Si se establece en "ninguno" o 0, la semilla será elegida
al azar. Esto también se puede configurar directamente a través de la línea de comandos
"seed", que anulará cualquier configuración de archivo rc. Este valor es un
sin firmar 64 bits entero.

pregen dungeon = incremental
Este ajuste determina el modo de generación de nivel para nuevos juegos, que
interactúa con la visualización aleatoria (ver la sección "juego visto" de la
manual). Para generar una mazmorra estable de una semilla, los niveles necesitan
para ser generado en un orden consistente. The recommended setting of
`incremental` generará niveles según sea necesario al entrar en un nuevo nivel
para que el juego siempre genere niveles en el mismo orden, potencialmente
ponerse al día al saltar niveles en el orden estándar.

Cuando se establece en 'verdad' o 'full', el juego pregenerará todo
mazmorra conectada al iniciar un nuevo personaje. Esto conduce a
determinista generación de mazmorras en relación con una semilla de juego particular,
y no tiempos de carga al entrar niveles, pero al costo de un juego lento
Empieza. Algunos servidores pueden desactivar la pregeneración completa para jugar en línea.

Cuando se establece en 'falso' o 'clásico', el juego generará todos los niveles en
nivel de entrada, como era la regla antes de 0.23. Dungeons no será estable
dada una semilla con esta opción.

supresión startup errores = falso
Si esto es falso, y se detecta un error mientras el juego comienza primero
(como un error en un archivo de configuración), subir una pantalla antes
cargando para mostrar estos.

El menú principal contendrá una advertencia de errores independientemente de esto
configuración, y no afecta lo que sucede con errores detectados como
El personaje está cargado.

mapa = ""
Esto se puede utilizar para elegir un mapa de sprint. Si se establece en un archivo rc, esto
valor salta el sprint mapa selección submenu. Las opciones válidas son:
dungeon sprint 1, the violet keep of menkaure, dungeon sprint mu,
dungeon sprint fedhas, sprint v, arena sprint, pitsprint, carnesprint,
y lineprint.

completamente aleatorio = falso
Es verdad elegir un combo aleatoriamente. Si alguna especie o trabajo es
establecido en 'viable', esto elegirá un combo de "recomendado"
caracteres, de lo contrario, elegirá combos al azar. Cuando
set, los menús de selección de caracteres se reemplazan con un impulso que
le ofrece un combo, y le permite aceptar o rerollar.

arena teams = ""
Establecer para preselegir equipos en modo arena. Cuando se establece, el modo arena va directamente
a la competencia.

2- Sistema de archivos.
====================

Las siguientes opciones de directorio pueden ser ignoradas o anuladas en algunos sistemas
y construye. Por ejemplo, no se pueden establecer en un archivo rc para jugar en línea. Ellos
también se puede configurar a través de la línea de comandos.

gate dir =
Un camino a ser utilizado para los caminos relativos al buscar otros
archivos. El valor predeterminado depende del sistema y
tipo de construcción. Esta opción también puede establecerse en el entorno 'CRAWL DIR'
variable. Cuando se establece, esto superará los tres directorios siguientes
opciones si han sido previamente establecidas.

save dir = saves/
Directorio donde se almacenan los ahorros y los huesos. Un camino relativo
se interpretará en relación con el valor de `crawl dir`.

morgue dir = morgues/
Directorio donde morgue descarga archivos (morgue*.txt y
morgue*.lst) así como archivos de volcados de caracteres están escritos. Un pariente
el camino se interpretará en relación con el valor de `crawl dir`.

macro dir =
Directorio para la lectura macro.txt.
Debe terminar con el delimitador del camino. El valor predeterminado para esto
depende del sistema y del tipo de construcción.

sonidos on = verdadero
(Requiere "Apoyo de sonido"; consulte la información de su versión)
Si es cierto, juega efectos sonoros en varias situaciones.

sonido ^= <regex confianza: se realizó a través de un archivo de sonido
(opción de lista ordenada)
Reproduce el archivo de sonido si un mensaje contiene regex. En caso de
múltiples regexes coincidentes, sólo el sonido del último regex
se juega. El regex no debe incluir comas ni colones.
Por ejemplo
sonido += LOW HITPOINT WARNING:sound\sounds2\danger3.wav
Hay ciertos regexes predefinidos, también, que pueden ser
usó para conseguir un sonido para desencadenar algo que no puede
de lo contrario se igualan usando una cadena regex. Por ejemplo,
usando FIRE PROMPT SOUND para el regex causará que Crawl
Toca un sonido cada vez que se abre el aviso de incendios. Check
sound.h para una lista completa de todos estos regex predefinidos
cuerdas.

sound file path =
Al reproducir sonidos (con la opción anterior), el contenido de
esta variable se adjunta al inicio de la ruta. Esto es
diseñado para hacer más fácil tomar una configuración en
varias plataformas. Estas dos líneas son equivalentes a las
ejemplo anterior:
sound file path = sound\sounds2\
sonido += LOW HITPOINT WARNING:danger3.wav

Ven a ver.

Disparado/ disparado. codicioso pickup automáticamente hace esto, pero
codicioso pickup smart no. explore stop pickup ignore
no afecta esta afección.

codicioso visited item stack: Pare cuando visite una pila de elementos
anteriormente desconocido para el jugador, incluso si la pila contiene
nada elegible para autopickup.

brillo Artículos: como artículos, pero sólo para artículos que son
brillante / correda / bordada / etc.

artefactos: como objetos, pero sólo para artefactos.

runas: como artículos, pero sólo para runas.

Puede especificar explícitamente los criterios de clasificación en el sort menus
opción como:
sort menus = true : art, basename, qualname, maldición, qty

Se compararán dos artículos sobre la base de los criterios de primera clase
donde difieren. Así que con la línea kind menus dada arriba,
si sólo uno de dos elementos diferentes es un artefacto conocido, será
enumerado primero, si ambos o ninguno son artefactos)

sus nombres de base; si los nombres de base son los mismos pero los calificados
los nombres son diferentes que comparará sus nombres calificados, y así
adelante.

Los criterios disponibles son:

* nombre de base:
Este es el nombre del tipo de artículo. El nombre base para todos
"una túnica +0", "una túnica bordadada" y "la túnica maldecida +2
Ponies es sólo "robe". El nombre base para ambos "un metal
Anillo y "un anillo de resistencia al fuego" son "ring".

* qualname:
El nombre del artículo sin artículos (a/an/the), cantidades,
encantamientos, o maldición-estado. Los nombres calificados para los
las túnicas descritas anteriormente son "robo", "túnica bordadada" y "robe"
de Ponies", respectivamente. Los nombres calificados para los anillos
descritos arriba son "anillo del cerebro" y "ring of fire
resistencia", respectivamente.

* nombre completo:
Este es el nombre del elemento que se muestra en los menús (incluyendo
(cuantidades, estado de maldición, etc.)

* dbname:
Clasificaciones basadas en el nombre interno único de un artículo que está vinculado
a su descripción. Los artículos con la misma descripción serán
agrupados.

* maldición:
Curse-status del artículo (si se sabe). Los artículos incursionados aparecen
primero.

* equipado:
Los elementos equipados aparecen primero.

* art:
Los artefactos identificados aparecen primero.

* ego:
Los objetos de ego identificados aparecen primero.

* Brillante:
Los elementos no identificados brillan/shiny/runed/etc aparecen primero.

* identificado:
Identified items show up before unidentified ones of the
mismo tipo. Un artículo es considerado como identificado una vez que tú sabe
su subtipo o ego.

* qty:
La cantidad para elementos apilables (como pergaminos, pociones,
etc.)

* ranura:
The inventory letter for items in inventory; irrelevant for
artículos en el suelo.

* Cargado:
Hace aparecer a los evokers con cargos disponibles antes de aquellos que
están vacíos; irrelevantes para todos los demás tipos de artículos.

* utilidad:
Ordenará los consumibles por su 'usocidad' actual en el
siguiente orden: Emergencia (por ejemplo: ?blinking, !heal wounds),
Bien (por ejemplo: ?acquirement), Normal, Peligroso (por ejemplo: !lignificación),
Mal (por ejemplo: la tortura mientras adora a un buen dios), y finalmente
permanentemente inútil.

Tú puede pedir un orden descendente por prefijo uno o más
clasificar los criterios con el título como:
sort menus = verdadero : nombre de base,

También puede solicitar la clasificación sólo para menús específicos:
sort menus = pickup: true
o
class menus = inv: true
(Los tipos de menu deben especificarse como nombre:, sin espacio entre
nombre y colon.)

en su inventario.
menú desplegable.
inv: Listas de inventario para cualquier comando (pero no para dejar caer
Temas).
cualquiera: Todos los menús; este es el predeterminado cuando no especificado.

Si defines el sort menus de forma diferente para dos o más selectores,
el último que coincida siempre tendrá precedencia, es decir, "nada"

spell slot ^= <regex
(Opción de lista ordenada, un valor por línea)
Cuando memorices cualquier hechizo que coincida con el regex, asignará
a la primera carta disponible en la lista. A + en la lista de
letras en "modo de escritura": todas las letras hasta el siguiente "-" son
considerado disponible incluso si ya se ha asignado; en ese caso, cualquier
hechizo ya asignado a esa ranura, siempre que no
coinciden con el mismo.

Si todas las letras de la lista están ocupadas para cada regex el hechizo
fósforos, utilizará el abc...xyzABC..XYZ. Si
hechizo coincide con múltiples regexes, sólo el primero se considera.

Ejemplos:
* si quieres que Apportation sea colocado en A:
spell slot ^= Aportación: A
* si quieres que el Armour de Ozocubu sea colocado en r normalmente o R si

hechizo slot ^= Armour de Ozocubu:rR
* si quieres que el primer hechizo "Bolt" sea colocado en un, incluso si
ya hay un hechizo no-bolt allí:
hechizo slot ^= Bolt:+a
* si desea cambiar la asignación por defecto de la ranura del hechizo
para utilizar letras mayúsculas: (situar esto después de todas las demás
orto slot lines):
spell slot += .*:ABCDEFGHIJKLMNOPQRSTUVWXYZ

menu colour ^= <match título: Utilizar instrucciones para mejorar el contenido:
(opción de lista ordenada)
Para menús y otras pantallas que imprimen elementos (y algunos otros casos; vea
lista abajo) esta opción le permite controlar el color que se utiliza
Mostrar una línea utilizando una expresión regular. Por ejemplo, el defecto
las opciones utilizan la siguiente línea:
menu color += lightmagenta:.*orb.*Zot
para destacar el orbe de Zot como púrpura en el menú de recogida de elementos. Para
ítems, la expresión regular también se comprueba contra diversos elementos
anotaciones como se describe a continuación. Por ejemplo, la siguiente línea:
menu color += amarillo:.*emergency item.*
causa elementos que obtienen la anotación `emergence item` (como pergaminos
de teletransportación) a destacar en amarillo en inventario y otros
menús de artículos.

Puede haber varias declaraciones en una lista, y también varios menu colour
líneas. Al utilizar varias líneas menu colour, el color del  first 
se aplica regex correspondiente; la opción es compatible con `^=`, que es útil
para anular los ajustes predeterminados sin construir la lista desde cero.
Para una lista de colores, consulte la opción de color en 5-a.

La parte de la `match ' especifica qué listados son afectados por la
colorear. Si especificas 'item', o salta completamente el <match título:
parte, entonces el patrón se aplicará a todas las categorías relacionadas con los artículos.
Los posibles valores relacionados con los artículos para el partido son:
inventario (inventario y menús de recogida y caída)
recoger (pantallas, y elementos en el suelo)
equipar (las [ y " pantallas)
use item (equip, unequip, read, quaff, etc)
tienda (pantallas)
stash (los resultados de Ctrl-F)
descripción (la pantalla \)
Estadísticas (artículos blindados/preparados en las estadísticas de pantalla principal
panel)
resiste (artículos en la pantalla %)

Los posibles valores no relacionados con el tema son los siguientes:
explicit `match`:
ayuda (el manual)
notas (la?: pantalla)

Los elementos son anotados con muchas etiquetas útiles que hacen el control del color
más fácil. Los elementos están anotados con su tipo de base (armour, arma, varita,
etc.) También están anotados con ciertas etiquetas que indican utilidad:
prohibido (Tu dios lo odiaría si utilizaste este artículo.)
emergency item (Este artículo es invaluable en emergencias.)
good item (Este artículo es generalmente un buen artículo.)
dangerous item (Usar este artículo puede ser peligroso.)
bad item (Este artículo es generalmente un mal artículo.)
inútil item (Este artículo no sirve para tú.)

así como una gama de otras propiedades:
equipados (Equipados artículos.)
ego (Se trata de una marca mágica ofensiva/defensiva).
artefacto (Para artefactos, identificados o no).
unrand (Para artefactos no aleatorios.)
melee (melee weapons, including magic staves.)
ranged (Ranged weapons.)
evoker (Evokers Elemental.)
identificado (El tema está completamente identificado.)
conocido (Tú reconoce el subtipo del artículo).
no identificado (Tú no reconoce el subtipo del artículo.)
regalo de dios (un artículo que te ha dado un dios.)
maldito (Maldito por Ashenzari.)

(Por supuesto, algunos de estos sólo se aplicarán a algunos menús inherentemente: solamente
los elementos transportados pueden ser equipados, sólo los artículos no cargados pueden ser
no identificado, etc.)

Al buscar los partidos de menu colour, estos prefijos se tratan como si
son prepended to the actual item name, for example:
identificado mal Artículo prohibido error una pieza de Xom
tablero de ajedrez (15/15)

`dat/defaults/menu colours.txt`.

Canales de mensaje de 3 k.
-----

Crawl se comunica a los jugadores con su ventana de mensajes. Cada mensaje
pertenece a uno de los llamados canales. El comportamiento de cada canal
se puede cambiar con la opción

canal.CHANNEL NAME = (COLOUR TENIDO mute ANTERIOR predeterminado TENIDO EN TENIDO Silencioso)

CHANNEL NAME puede ser uno de estos:
llano = texto regular (y cosas "no cubiertas")
prompt = impulso de entrada al jugador
Dios = mensajes de los dioses
duración = mensajes sobre hechizos/efectos de carácter que se agotan
peligro = amenazas graves a la existencia del personaje
advertencia = varias otras advertencias
recuperación = recuperación de enfermedades/estatal pérdida/envenenamiento
hablar = monstruos hablando
talk visual = monstruo realizando alguna acción que el jugador ve
timed portal = mensajes de timeout portal
sonido = otros sonidos
intrinsic gain = nivel/stat/especie aumentos de potencia
mutación = ganancia o pérdida
monstruo spell = mensajes sobre monstruos gesturing or casting hechizos
monstruo enchant = mensajes relativos a los encantamientos de monstruo
monstruo warning = monstruos que vienen a la vista
amigo spell = como monstruo spell, pero sólo para monstruos amistosos
amigo enchant = como monstruo enchant, pero sólo para monstruos amistosos
friend action = otras acciones de monstruos amigables
monstruo damage = mensajes diciendo lo dañado que es un monstruo
monstruo target = mensajes que marcan al monstruo como un objetivo (no utilizado)
destierro = mensajes sobre el destierro y el destierro al abismo
equipo = mensajes que indican el equipo gastado / blindado
suelo = mensajes al mirar o caminar sobre un objeto de piso
multiturn = indica acciones largas (desgastando armadura, diseccionando etc.)
examinar = mensajes de examinar su entorno
examine filter = mensajes aburridos de examinar su entorno
diagnóstico = mensajes de depuración
error = mensajes de error
tutorial = mensajes del tutorial
orb = mensajes sobre el Orbe de Zot
hell effect = mensajes sobre los efectos ambientales de los infiernos
dgl message = mensajes de los vigilantes (sólo juegos de consola en línea)

Las opciones del canal son
mute = no mostrar mensajes del canal (peligroso, ten cuidado!)
default = canal de encendido a su esquema predeterminado
alternativa = canalizar a su esquema alternativo "courioso"
on = same as default
llano = hacer el canal el mismo color que el canal "plain"
(no hacer nada tonto como "mute" si claro == mudo,
aunque)
fuera = igual que simple

COLOUR puede ser cualquiera de los colores descritos en la sección 5-a (opción de color).

Los únicos canales multicolores actualmente son monstruo damage y dios. Todos
otros canales están predeterminados.

Crawl puede registrar automáticamente ciertos eventos durante el juego. Puedes leer
estos en los archivos de basura o morgue. A continuación se presentan las opciones para ajustar esto
comportamiento.
Los siguientes eventos se han identificado:
- Ganar o perder un nivel
- Entrando en un nivel de mazmorras por primera vez
- Memorizar un hechizo de mayor nivel que cualquier aprendido antes
- Ser adorador de un dios
- Abandonar a un dios
- Ser puesto bajo penitencia y ser perdonado
- Recibir un regalo de un dios (excepto Xom)
- Muerte de un aliado orcish (cuando adoraba Beogh)
- Ser capaz de invocar un poder divino por primera vez
- Recoger una runa, el Orbe de Zot, o un artefacto por primera vez.
- Identificando objetos.
- Matar OOD o monstruos únicos (ver abajo)
- Alcanzar niveles críticos de HP (ver más abajo)
- Obtención o pérdida de mutaciones
- Alcanzar niveles significativos en una habilidad (ver abajo)
- Dying
Puede utilizar el comando ':' para agregar notas manualmente.

Prefijos añadidos manualmente notas con la cadena dada, para hacer
más fácil de encontrar. Actualmente no es posible añadir un

6-a Incluyendo archivos de lua.
-----

Los archivos Lua son scripts que pueden proporcionar comandos existentes con un nuevo
significar o crear nuevos comandos (para ser utilizados en macros). Para usar Lua
Archivos, Crawl necesita ser compilado con soporte para scripts Lua usuario.
Tú puede comprobar para ver si su Crawl tiene soporte de Lua golpeando ?V en el juego.
La lista de características Las pantallas de Crawl deben incluir "Lua scripts de usuario".

Los archivos de lua se incluyen utilizando la opción lua file (un archivo por línea):

lua file = <path/name.lua

Las funciones de lua pueden ser macroeconadas en el juego estableciendo la acción macro a "===",
seguido del nombre de la función.

El mago-modo Lua intérprete ( aportar^T), la primera vez que se invoca,
cargar todos los archivos que se especifican con la opción terp file:

terp file = <path/name.lua

El Lua en estos archivos tendrá acceso a todos los internos de Crawl Lua
(es decir, se ejecutará en el contexto de dlua, no clua).

6-b Ejecutando lua inline.
-------

El código lua se puede utilizar directamente en su init.txt/.crawlrc. Puedes
ejecutar el código Lua usando la siguiente sintaxis.

: Línea única de código lua

* Posiblemente multilínea
Código de lua

{ Posiblemente multilínea
Código de la lua

En los casos segundo y tercero, la restricción es que el delimitador
caracteres aparecen al principio y al final de una línea, respectivamente.
La diferencia entre el nombre de usuario y {} es cuando el código se ejecuta.
Código {}, se ejecuta inmediatamente. Otro código Lua se ejecuta solamente
después de que el archivo entero se lee.

Ejemplos:

Imprimir un mensaje de bienvenida
: gate.mpr("Hola " .. tu nombre())

c)
-- Another welcome message (lua code uses lua comments)
gate.mpr("Hola allí")
■

{}
función ch autopickup(it) [ ... cuerpo omitido ... ] final
}

6-c Opciones condicionales.
----------

Puede utilizar Lua para incluir selectivamente partes de su init.txt (basado
en el tipo de caracteres, por ejemplo) usando la misma sintaxis.

Ejemplo:

Si tú.race() == "Mamá" entonces
autopickup = $?+"/
:
autopickup = $?+"/!
: final

Las opciones pueden ser referenciadas por lua a través de "options.option name". Para
ejemplo:

:if string.find(options.autopickup, "!") entonces
# Haz algo aquí si las pociones están incluidas en el autopickup
:end

"options.option name" puede incluso ser utilizado para opciones que se arrastran a sí mismo
no reconoce. Esto se puede combinar con opciones de configuración en
línea de comandos (ver sección 0-b) para utilizar la línea de comandos para controlar
condicionalización de opciones en los archivos de opciones. Por ejemplo, en el
línea de comando que podría establecer la opción "foobar" con "-extra-opt-first
foobar=true", y luego hacer:

:si opciones.foobar entonces
# Hacer cosas aquí
:end

6-d Opción condicional caveats.
---

Tenga en cuenta que ninguna de las opciones enumeradas en "Starting Screen" (sección 1)
puede establecerse condicionalmente. Esto es porque los archivos de opciones son
leer en dos ocasiones: una vez antes de la creación de carácter con Lua girado
y una segunda vez después de la creación de carácter con Lua encendido. Si
intenta establecer una opción de pantalla inicial condicionalmente entonces el valor
más lejos en el archivo de opciones se utilizará independientemente de qué
condiciones establecidas.

La caveat anterior también se aplica a la opción "wiz mode". En lugar de
wiz mode condicionalizado, se puede añadir a la línea de comandos
"-extra-opt-last wiz mode=yes" para hacer que cualquier nuevo juego comience en wizard
Modo.

7- Opciones recomendadas para jugadores ciegos.
================================================================================================================================================================================================================================================================

Se recomiendan los siguientes ajustes para facilitar el uso de la consola arrastre
con un lector de pantalla.

Desactivar animaciones y ver demoras para que la pantalla actualice inmediatamente después de cada
comando:
view delay = 0
travel delay = 1
rest delay = 1
use animations = 0

Colocar mensajes en la parte superior de la pantalla y eliminar el mensaje condensando para
lectura de pantalla más fácil:
mensajes at top = verdadero
clear messages = true
msg condense short = false

Hacer que el cursor del jugador sea más fácil de rastrear con la lectura de la pantalla, estableciendo una costumbre
símbolo para el jugador:
use fake player cursor = false
show player species = false
mon glyph += jugador:x263A

Utilice los clics de entrada del ratón para el automóvil (ver la discusión del ratón input
opción para configuraciones terminales:
mouse input = true

Permitir auto-travel/move para avanzar un paso incluso cuando los monstruos están presentes:
travel one unsafe move = true

Coordenadas relativas al jugador en la pantalla ^x:
monstruo item view coordinates = verdadero

La pantalla ^x puede enumerar un conjunto personalizable de características controladas con regularidad
expresiones. Por ejemplo, incluir todo el agua en ^x set:
monstruo item view features += agua

Marque el camino tomado por auto-travel y el horizonte de exploración con especial
caracteres:
show travel trail = true
característica += ruta de viaje {x25AA}
función += explorar horizonte {x25AB}

Establece un glifo separado para escaleras y transportadores no visibles:
característica += escalera de piedra que conduce {
característica += escalera de piedra que conduce { prenda,x25B8}
característica += transportador {xA9, x25CE}

Visita cuadrados con pilas de elementos al explorar para imprimir todos los nuevos
objetos encontrados:
explore greedy visit += pilas
